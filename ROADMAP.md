# Roadmap

## Now — validate the v0.1 source and macOS installer candidate

- Run the configured macOS, Windows, and Ubuntu GitHub Actions matrix on the public commit.
- Exercise the local arm64 and x64 DMG packaging paths without committing generated artifacts.
- Obtain protected Developer ID/notary inputs and validate signing, notarization, stapling, Gatekeeper, and downloaded-asset checksums before publishing a macOS installer.
- Complete physical macOS camera/permission/tray/alert/lock/sleep checks.
- Recruit a Windows 11 x64 volunteer for the versioned checklist; keep the claim at “CI-tested / physical verification wanted” until then.
- Record screen-reader, keyboard, 200% zoom, contrast, and reduced-motion results on supported platforms.
- Publish current resource/soak measurements and privacy-preserving formative alert-helpfulness findings.

## Next

- Fix evidence-backed usability, accessibility, driver, and alert-noise issues from v0.1 testing.
- Add independently verified support tiers without turning CI or anecdotes into physical/health claims.
- Improve contributor fixtures and translations without accepting personal camera material.

## Later / exploring

- Multiple explicit desk profiles.
- Day/week aggregate history.
- Optional break reminders and local schedules.
- Community localization and additional accessibility modes.

These are not commitments or dates. A direct macOS DMG is in scope. Accounts, backends, telemetry, automatic updates, App Store distribution, Windows/Linux installers, social competition, medical claims, raw recordings, and plugin systems remain out of scope for v0.x.
